Tool Center Point (TCP)
Saved tool configs
Save the tool defined above (manual or teach‑solved) as a named config. Click Apply to load one onto the robot; ✕ to delete. Blank/duplicate names auto‑number so every save is kept. Shared across UIs and persists across restarts.
4‑point teach (tool tip → one fixed point)
Jog the tool tip onto the same fixed point from ≥4 different orientations; capture each. Solves the offset by least squares.
Orientation teach (point along the tool axes)
Do this after the offset. Keep the flange orientation FIXED (translate only, base‑frame jog): capture Ref, then jog the tip a few cm along the tool's pointing/approach axis and capture +Z. Optional: jog along the tool's +X and capture +X (skip for a round marker — roll is free).
Physical‑contact detection
Sensorless contact detection: the drives' measured joint torque is compared against the dynamic model (gravity + inertia + friction + payload). An unexplained external torque means the arm touched something → protective stop. This is a software monitor and does not replace the hardware E‑STOP / STO.
Detector
Off by default — turn on only after tuning on your arm, or it may nuisance‑stop the drives. The hardware E‑STOP/STO protects the arm regardless.

Fully automated from joints‑0: goes to the joints‑0 home, moves to a fixed pose holding the marker perpendicular to the base (pen straight down) over the front of the robot, descends straight down until it touches the table (coarse then fine, contact‑sensed so the tip isn't damaged), then builds & activates a level writing plane at that height.
